HTML5 Style & Coding Conventions

Code must be Smart & Future Proof:

Code must have a consistent user of style, this makes code to easier understand written code.

In future XML readers or other readers may try to read documents made, meaning

Use of Correct Document Type:

Use !DOCTYPE html.

Rather than: !doctype html.

Use Lower Case Element Names:

HTML5 allows the mixing of uppercase and lowercase letters in element names.

Lowercase element names are recommended because:

  • Mixing uppercase and lowercase names is bad.
  • Developers normally use lowercase names (like XHTML).
  • Lowercase looks cleaner.
  • Lowercase are easier to write.

All Elements must be closed:

In HTML5, not all elements need to be closed, however it's recommented.

A paragraph element doesn't need to be closed, but it's highly recommended.

Close empty HTML elements:

Elements such as meta or hr don't require closing, but it's recommended.

Use Lower case Attribute names:

Like elements, HTML5 allows for the mixing of uppercase and lowercase attribute names.

Quote Attribute Values:

HTML5 allows Attribute values without quotes.

Quotes are highly recommended because:

  • Developers normally use quote attribute values (like in HTML).
  • Quoted values are easier to read.
  • Quotes MUST be used if the value contains spaces.

Image Attributes:

Images must always contain the alt attribute. This attribute is important if the image can't be displayeed.

The width and height must also be defined. By doing this it can reduce flickering as the browser can reserve space for the image before loading.

Spaces and Equal Signs:

HTML5 allows spaces around equal signs. Space-less is easier to read and groups entities better together.

Avoid Long Code Lines:

When using a HTML editor, it's inconveinent and more time consuming to scroll left & right to read the HTML code.

To avoid this, lines of over 80 characters should be avoided.

Blank Lines & Indentation:

Blank lines shouldn't be added without reason.

For readability, blank spaces are good for separating large or logical code blocks.

According to W3, rather than tabs, two spaces should be used, however it's down to the organisation/individuals' discretion.

Unnecessary blank lines & indentation shouldn't be used. It's not necessary to indent every element.

Omitting html and body Tags:

Although not recommended, the html and body element tags can be omitted.

Omitting head:

Again, not recommended, the head tag can be omitted to reduce complexity. This shouldn't be done as it's unfamiliar to developers and needs time to be established as a guideline.

Meta Data:

The title element is required in HTML5, so must be meaningful.

To ensure proper interpretation and correct search engine indexing, both the language and the character encoding should be defined as early as possible in a document.

Setting the Viewport:

HTML5 introduced a method to let web designers take control over the viewport, via the meta tag.

The viewport is the user's visible area of a web page. It varies with the device, and will be smaller on a mobile than on a computer screen.

The meta viewport should be included in every page.

A meta viewport element gives the browser instruction on how to control the page's dimensions and scaling.

The width=device-width sets the width of the page to follow the screen-width of the device(which will vary).

The initial-scale=1.0 part sets the initial zoon level when the page is first loaded by the browser.

Style Sheets:

Simple syntax for linking to style sheets (the type attribute is not necessary).

Short rules in CSS can be written compressed {element:value;}

Long rules with many element and values should span multiple lines:

Use Lower Case File Names:

File servers such as Apache & Unix are case sensitive, wherease others such as Microsoft & IIS are not case sensitive. Because some servers are and some aren't, naming must be kept very extremely consistent.

Differences Between .html and .html:

There's no difference between .html and .html extensions. Browsers will both be treated as HTML.

The differences are cultural, .htm is where early DOS systems were limited to 3 character extensions. .html is from Unit where there isn't that limitation.

Technical Differences

When a URL does not specify a filename (like https://www.w3schools.com/css/), the server returns a default filename. Common default filenames are index.html, index.htm, default.html and default.htm.

If your server is configured only with "index.html" as default filename, your file must be named "index.html", not "index.htm."

However, servers can be configured with more than one default filename, and normally you can set up as many default filenames as needed.

Anyway, the full extension for HTML files is .html, and there's no reason it should not be used.
