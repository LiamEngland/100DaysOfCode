HTML URL Encoding Reference

URL encoding coverts non ASCII characters into a format that can be transmitted over the internet, here is a reference to all URL encodings

Browsers will request pages from web servers by using a URL, which the domain's server will return whichever file or page is located on that path.

A web address follows these syntax rules: scheme://prefix.domain:port/path/filename

  • Scheme - Defines the type of Internet service (Most commonly HTTP or HTTPS)
  • Prefix - Defines a domain prefix (default for HTTP is WWW
  • Domain - Defines the Internet Domain Name (like liamengland.net)
  • Port - Defines the port number ath the host, (default for HTTP is 80)
  • Path - Defines a path at the server (If omittedL the root directory of the site)
  • Filename - Defines the name of a document or resource (For example; index.html)

URL Encoding converts non-ASCII characters into a format that can be transferred over the internet, using the reference previously mentioned.

URLs cannot contain spaces, in there place a space is replaced with a plus sign or %20.
